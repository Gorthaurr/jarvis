## 1. Топ-проблемы по приоритету

### P0 — Каскад «чужая речь → задача → слепой авто-реплей макроса»

Окно диалога после wake принимает чужую речь (91+94 перехода тихо/шумно за 13-е) → роутер маркирует
почти всё «задачей» (51 из 74 ходов) → гибридный recall на шумном e5-small пробивается генерик-навыками
(sim 0.82–0.89) → **авто-реплей жестов в активное окно**. Из 15 авто-реплеев за 2 дня легитимных ~5:

- 13.07 10:20:41: «Я тебе пизды сейчас дам, уёбище…» → recall `zakryt-prilozhenie` (0.831) → реплей во
  время пика героя в Dota; «задача» ждала аренду ввода 60с, отменена вручную ($0.18).
- 13.07 13:16:53: «я просираюсь от чернослива» → тот же навык (0.821) → реплей ok.
- 13.07 16:48: разговор о базе данных → ×2 реплей 14-шагового `pisat-nadezhnyi-makros` («нажал S куда
  попало» — признал сам Джарвис) + ×1 «закрыть приложение».
- 14.07 12:06:54: инструкция «Не забывай уточнять у меня, кому писать» → recall (0.847) → реплей ok.
- «Нет, заткни» стало задачей; её повторы получили ×2 «Уже делаю, сэр» (дубль-гейт на стоп-команде).

Спикер-гейт при этом выключен (`JARVIS_SPEAKER_GATE=0`), хотя движок готов (`ready:true, voices:1`).
*Частично закрывается инкрементом «строгий wake в шуме» (вход через окно разговора); recall-порог
авто-реплея и гейт «это вообще команда?» — отдельные дыры.*

## 3. Рекомендации (порядок = приоритет)

1. **Гейт авто-реплея (P0):** порог sim ≥ ~0.92 + обязательный глагол действия + precondition на первый
   шаг + запрет авто-реплея мета-навыков и реплея из окна диалога без wake / на conversational-классе.
2. **Адресация в окне диалога (P0):** спикер-гейт (готов) или классификатор «это мне?»; стоп-лексика
   «заткни» → stop ДО создания задачи; брань/эмоции → conversational без аренды ввода; фрагмент без
   глагола → clarify. *(Вход через окно частично закрыт инкрементом «строгий wake в шуме».)*
3. **Earcon на sync-first (P1):** тик на 1.5–2с без первого чанка TTS — закрывает кластер «10с тишины».
4. **Кеш-экономика (P1):** не эскалировать при остатке <3–4 раундов до потолка; липкость модели до конца
   задачи; стабилизировать кеш-ключ step 0→1 recall-задач; single-flight холодного прогрева; аудит
   префикса 40–50K → ~25K; дешёвый профиль разговорных ходов.
5. **Дизамбигуация получателя (P1):** ≥2 тёзки → обязательный clarify до telegram_send; негативный
   сигнал в resolution-memory.
6. **Доказательство отправки (P1):** «набрал текст → Enter» терминалится только по дифу состояния
   (поле опустело / текст в ленте); после успешного skill.execute — мини-бюджет verify 2–3 раунда.
7. **Garbled-детектор (P2):** чисто-кириллический короткий фрагмент без глагола + низкий confidence →
   $0-clarify.
8. **Дубль-гейт/steer (P2):** окно recentTerminal («уже сделал <60с назад»); reject-маркеры
   «не ушло/не сработало» → steer, не новая задача.
9. **Handshake-гигиена (P2):** буфер audio.frame до server-ack; глушить client.state в resume-окне;
   буферизовать терминальную фразу до reconnect.
10. **Наблюдаемость:** queueWaitMs+title в metrics.jsonl; barge-in-строка в серверный лог; кодировка
    title в admission; причина в «РЕЧЬ ПОТЕРЯНА».
