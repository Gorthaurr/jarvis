## ГЛАВНАЯ ЗАДАЧА СЛЕДУЮЩЕЙ СЕССИИ (запрос владельца)

**Концепт «своя среда»: дать модели делать НА ПК всё — быстро.** Владелец спросил: «мб стоит придумать
свою среду и концепт, который позволит модели делать абсолютно всё на компе». Согласованное направление
(обсуждено в сессии 2026-07-10, владелец ждёт проработку):

1. **НЕ песочница/VM** — Джарвис живёт на личном ПК владельца (его логины/файлы/игры); изоляция убивает смысл.
2. Ядро — 4 сдвига: (а) наблюдение = ПОДПИСКА (WinEvent/UIA-events/watchers → «разбуди, когда появится
   окно X»), не опрос скриншотами; (б) **world model** — живое дерево окна→элементы→состояние на сервере,
   обновляемое событиями; инструменты = дешёвые запросы к нему; (в) **persistent Джарвис-SDK** — живой
   Python/PS-интерпретатор с СОСТОЯНИЕМ между вызовами и API вида `jarvis.window("Dota").click(...)`,
   `jarvis.wait_for(text="Принять")`: один LLM-раунд = целая процедура, не один клик (ГЛАВНЫЙ скачок,
   единственное, что реально выходит за рамки Волн 2-3); (г) среда наращивает свой API — выученные
   HERMES-процедуры компилируются в именованные функции SDK (`jarvis.dota.accept()`).
3. (а)-(б) = радикализация Волн 2-3 из HARNESS_PLAN; (в)-(г) — новое. Безопасность — capability-модель
   (явное намерение владельца = токен на необратимое), не урезание мощи.
4. Честные пределы, которые среда не чинит: реалтайм-рефлексы игр, качество STT, недетерминизм чужих GUI.

Старт: либо концепт-документ в docs/ (этапы/риски), либо СНАЧАЛА deep-research по persistent-execution
и world-model агентам 2025-26 (Claude Agent SDK, CUA-harness'ы) — владелец выбор не озвучил; рисёрч
надёжнее (концепт на фактах, не на голове).

  (комплимент стоил $0.19 и 8 раундов).
- Мелочь из ревью (не чинилось, низкий приоритет): greeting/watch-checker не учитывают траты в
  SpendGuard (существующая дыра); дефолтная Steam-библиотека сканится дважды (лишний IO);
  presence-строка не шлётся при пустом окно-снимке; лексический CANCEL_WORDS без «останови»
  (осознанно: конфликт с медиа-паузой).
- Мониторинг keyterm: если Deepgram «замолчит» (0 Results при идущих кадрах) — убрать
  `DEEPGRAM_KEYTERM=1` из `.env` (подозрение описано в deepgram.ts:112).
