# Следующая сессия: ЖИВАЯ РЕЧЬ — человечное произношение через ElevenLabs

> Цель (запрос пользователя): сделать речь Джарвиса максимально человечной — естественные **паузы,
> ритм, интонация, дыхание**, всё произношение как у живого человека, а не диктора-автомата.
> Работаем с тем, что МЫ отправляем в ElevenLabs (текст + voice_settings + модель), не с самой сетью.
> Финальная приёмка — ТОЛЬКО на слух (как всегда с голосом).

## Что сейчас (база, чтобы знать, что менять)
Поток: ответ LLM → `verbalize()` → пофразно (`SentenceChunker`) → ElevenLabs → mp3 клиенту.
- **`verbalize()`** [apps/server/src/brain/verbalize/index.ts] — чистит markdown/URL/эмодзи; числа/время/
  валюту/телефоны → слова; схлопывает пробелы. **Прозы/пауз/интонации НЕТ — текст плоский.**
  ГРАБЛИ: `collapseWhitespace` превращает `\n` → «. » — может рвать ритм искусственными точками.
- **TTS** [apps/server/src/integrations/elevenlabs.ts]: модель `eleven_multilingual_v2`; `voice_settings`
  = {stability 0.6, similarity 0.8, **style 0.0**, speed 0.95, speaker_boost}; `apply_text_normalization:"off"`
  (нормализуем сами); `mp3_44100_128`; **НЕ стримится** (целый mp3 на фразу). Всё env-тюнится:
  `ELEVENLABS_MODEL/STABILITY/SIMILARITY/STYLE/SPEED/SPEAKER_BOOST`.
- **Чанкинг** [apps/server/src/brain/nlu/sentences.ts] — режет реплику на предложения для пофразного синтеза.

## Рычаги человечности (по убыванию эффекта)
1. **Паузы — пунктуацией и `<break>`.** ElevenLabs ведёт паузы/интонацию пунктуацией (`,` микропауза;
   `…` задумчивость; `—` пауза; `?`/`!` интонация) и тегом `<break time="0.4s"/>`. Поддержка `<break>`
   ЗАВИСИТ от модели — ПРОВЕРИТЬ на multilingual_v2 (на части моделей игнорируется; v3 — иначе).
   → Новый **humanizer-слой** после `verbalize`: вставлять естественные паузы на границах клауз,
   «…» для хеджа/раздумья, короткую паузу перед важным, дробить длинные периоды. БЕЗ перебора.
2. **Естественная формулировка (персона/LLM).** Сам текст: разговорные обороты, сокращения, короткие
   клаузы, живые связки — тогда ElevenLabs даёт «человеческий» материал. Правка persona.md: «пиши как
   говоришь — живой ритм, короткие клаузы, естественные связки» (но без воды и БЕЗ однотипности — см. v15-17).
3. **voice_settings под живость.** stability 0.6→~0.40-0.45 = выразительнее/вариативнее (человечнее), но
   риск «гуляния»; style 0.0→~0.1-0.2 = эмоция; speed по контексту. Можно контекстно (спокойный отчёт vs
   живая болтовня) через режимы §11 (getVoiceOpts уже есть). A/B на слух, env-тюнинг без перекомпиляции.
4. **Модель eleven_v3** (если доступна аккаунту) — самая выразительная + **audio tags**: `[breath]`,
   `[sigh]`, `[laughs]`, шёпот, эмфаза. Минус: выше латентность, иная система стабильности, не всегда для
   realtime. Вариант: v3 для НЕ-realtime реплик (фоновые итоги §20) или гибрид; multilingual_v2 — текущий баланс.
5. **Дыхание/маркеры — ДОЗИРОВАННО.** «ну», «так…», «хм», вздох перед длинным. Лучше через v3 audio tags,
   чем словами. Пользователь НЕ любит однотипность/«клоунаду» — цель естественность, не театр.
6. **Чанкинг по смыслу.** Границы фраз для стрима = ритм; резать по клаузам/паузам, не середину мысли.
7. **Нормализация-добивка.** verbalize неплох; добить аббревиатуры/единицы/англицизмы (чтобы не читалось
   по буквам / по-английски).

## План
1. Записать пару реальных ответов, послушать — что именно звучит «не по-человечески» (ритм? паузы? ровность?).
2. **Humanizer** — чистая тестируемая функция (после verbalize / на пофразном пути): пунктуация-паузы +
   `<break>` на границах + «…» для хеджа + дробление длинных. Покрыть тестами (как verbalize).
3. Проверить `<break>`/SSML на multilingual_v2 (доки ElevenLabs). Слабо → опираться на пунктуацию или v3.
4. Тюнинг voice_settings (env уже есть) — A/B на слух, найти «живую» точку; возможно контекстная по режимам §11.
5. Оценить eleven_v3 + audio tags (доступность в аккаунте, латентность); гибрид realtime/не-realtime.
6. persona.md — «живой ритм формулировок» (без воды/однотипности).
7. Приёмка — на слух пользователя.

## Грабли / НЕ ломать
- `apply_text_normalization:"off"` — мы нормализуем сами; `<break>`/SSML может конфликтовать с `off` на
  части моделей — проверить, при необходимости включать нормализацию точечно.
- НЕ сломать realtime-стриминг и latency-фиксы (паузы удлиняют АУДИО, не TTFT — это ок, но не блокировать первый звук).
- НЕ переборщить с маркерами/паузами (пользователь не любит однотипность и «клоунаду»).
- multilingual_v2 ограниченно понимает SSML; v3 — другой контракт. Сверять с АКТУАЛЬНЫМИ доками ElevenLabs.
- Связано с памятью [[project_jarvis_voice]] (voice_settings, модель, рус-нативные голоса, realtime-стек).
