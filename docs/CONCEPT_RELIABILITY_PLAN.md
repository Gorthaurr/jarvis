# Концепт + 100% попадание + скорость — план (аудит 2026-06-20, 70 находок)

> Требование Антона: ВСЕ возможности (игры, монтаж, мессенджеры, файлы, система, браузер) работают
> **по концепту** (модель водит ОБЩИМИ инструментами, сама думает — НЕ хардкод под сценарий) И
> **со 100% попаданием**, и при этом **быстро** (помнит лучший способ из прошлых раз).

## Четыре столпа (примиряют гибкость + надёжность + скорость)
1. **Verify-loop** — ОБЩАЯ обёртка: действие → наблюдай РЕАЛЬНЫЙ исход → «достигнуто?» → повтори(bounded) → честный итог. Надёжность.
2. **Зрение-подложка** — `screen_capture` в петле там, где нет a11y/DOM (игры, canvas, нативные, видеоредакторы). Универсальность.
3. **Опытная память (плейбук)** — сохранять ЛУЧШИЙ верифицированный способ (с измеренной ценой), recall→fast-replay→verify→self-heal. Скорость.
4. **Фоновая оптимизация** — на простое перепробовать альтернативы частых задач, апгрейдить плейбук. Не костенеет.

## Что показал аудит (70 подтверждённых находок)
- **Концепт почти ок:** 48/70 уже `general`. Болят **12 hardcoded** (свернуть per-site) + **10 missing** (дыры).
- **Надёжность — главный рычаг:** **20 `assumed-ok` + 11 `none` = 31 место рапортует успех без сверки.** Их закрывает ОДИН общий verify-loop, не 31 заплатка.
- **Ключевой структурный корень:** в основном agent-loop нет «посмотрел→действовал→посмотрел». persona ДЕКЛАРИРУЕТ «Verify-after-act is LAW», но (а) модель не всегда может сверить — НЕТ инструментов наблюдения; (б) skill-runner умеет post-condition только по a11y (бесполезно для canvas/игр/видео).

## Дыры (10 missing) — что создать
- Мессенджеры: WhatsApp/Discord/Slack (через общий `web_*`, не userbot).
- Наблюдение: `system_volume get` (readback), media-playback-state, post-write verify для office, baseline-screenshot для сравнения, network-roundtrip confirm.
- Зрение: авто-fallback на `screen_capture` при промахе a11y; явная persona-дока «когда зрение».
- Игры: контекст игрового состояния для мультишаговых последовательностей.

## Хардкод (12) — свернуть в общий путь
- Telegram CDP-селекторы (`.peer-title`/`.chatlist-chat`) + Я.Музыка `play/pause/next` в `pageActInPage` → общий инспектор/зрение «клик по смыслу».
- `system_volume`/`media`/`input`/`clipboard` помечены hardcoded в смысле «без сверки» → фикс = readback/verify, не де-хардкод.

## Риски / не ломать
- НЕ live-проверяемое в этой среде: PС-действия (громкость/ввод/игры), реальная отправка (нет залогиненного stealth-Chrome). Структурные P1/P2 требуют живого прогона ДО «готово».
- Держать зелёным сьют (server 637 / shared 26 / client 113). Persona-кеш §15 рушится на правке персоны — бампать version.
- Необратимый финал (отправка/заказ/удаление) НЕ бенчмаркать повтором — перебираем только идемпотентный путь.
